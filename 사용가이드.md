# RFP 제안서 작성기 — 상세 사용 가이드

> 공공 RFP 문서를 분석하여 제안서 초안을 작성하는 도구입니다.
> AI가 사업 분석부터 전략 토론, 장표별 본문 작성까지 자동화합니다.

---

## 📦 설치 및 실행

### 첫 실행
1. 폴더의 **`제안서작성기실행.bat`** 더블클릭
2. 처음 한 번만 Python 3.12 + 필요 패키지가 자동 설치 (5~10분, ~500MB, 인터넷 필수)
3. 설치 완료 후 자동으로 브라우저(`http://localhost:8501`)가 열림

### 두 번째 이후
- 같은 `제안서작성기실행.bat` 더블클릭 → 즉시 실행

### API 키 설정
- 폴더 안 **`.env`** 파일을 메모장으로 열기
- `OPENAI_API_KEY=sk-...` 부분에 본인 키 입력 후 저장
- 키 발급: https://platform.openai.com/api-keys
- 키 변경 후엔 앱 재시작 (콘솔 닫고 다시 실행)

### 종료
- 검은 콘솔 창 닫기 (또는 Ctrl+C)

## 🚦 전체 워크플로우

```
1. RFP PDF 업로드 (사이드바)
       ↓
2. 자동 파싱: 요구사항 / 목차 / 사업 개요 추출
       ↓
3. [선택] 목차 구성 탭: 슬라이드 정리, 요구사항 매핑 확인
       ↓
4. [선택] 💡 기능 아이디어 도출: 핵심 주제별 Agentic 채팅
       ↓
5. 🎯 전략 수립 탭: 챕터 선택 → 토론 → 장표별 초안 작성
       ↓
6. 📥 마크다운 다운로드 → Claude/GPT에 PPT 변환 요청
```

소요 시간 (1개 챕터 기준):
- 사업 분석/요구사항 파싱: 2~5분
- 챕터 전략 토론 (2라운드): 2~3분
- 장표별 초안 작성 (5개 슬라이드 × 5소제목): 5~10분
- **합계: 15~25분 / 챕터**

비용 (대략):
- RFP 1건 파싱: $0.5~1
- 챕터 1개 작업: $2~4
- 전체 (5~6챕터): **$10~25 / RFP 1건**

## 📂 사이드바

### 1. 데이터 입력

**📄 PDF 파싱 탭**
- RFP PDF 업로드 → "파싱 시작"
- 자동 추출:
  - 요구사항 (ID + 카테고리 + 명칭 + 정의 + 세부내용)
  - 목차 (장-절 구조)
  - 사업 개요 (사업명/배경/필요성/목적/목표/범위/기간)
  - 슬라이드 ↔ 요구사항 자동 매핑

**📥 엑셀 로드 탭**
- 이전에 다운로드한 엑셀 (요구사항 + 목차 + 사업 개요 시트) 재업로드
- LLM 호출 없이 즉시 로드 → 빠른 재개

**💾 세션 탭**
- 현재 작업 전체 (전략 토론, 장표 초안, 채팅, 사업 개요 등) JSON 다운로드
- JSON 업로드해서 복원 가능
- **권장**: 30분 이상 작업 시 중간 저장

### 📋 사업 개요
- 자동 추출된 사업 정보가 expander로 표시
- **편집 가능** — 추출이 부정확하면 직접 수정
- 모든 AI 호출에 컨텍스트로 자동 전달됨

### 진행 현황
- 슬라이드 완료율 progress 바
- 미커버 요구사항 카운트 (자세한 건 요구사항 탭에서)

## 🗂️ 목차 구성 탭

### 📷 이미지/VLLM으로 목차 추출
- 텍스트 추출이 부정확한 경우 (이미지·표 위주 RFP)
- RFP PDF 업로드 + 페이지 입력 (예: `30-35, 40`)
- VLLM(gpt-5.4)이 이미지 분석 → 슬라이드 + 요구사항 매핑 자동 생성
- **추가** 또는 **대체** 모드 선택

### 상단 액션
- **➕ 슬라이드 추가**: 빈 모달 열림
- **🔄 요구사항 재매핑**: 전체 슬라이드의 요구사항 매핑을 LLM이 다시 수행
- **⬇️ 엑셀 다운로드**: 사업개요 + 요구사항 + 목차 3시트로 export

### 슬라이드 목록
- 각 행: `상태아이콘 챕터-섹션 · 제목 + 연결 요구사항 미리보기`
- **✏️** 클릭 → 편집 모달
- **🗑️** 클릭 → 즉시 삭제

### 편집 모달 (`st.dialog`)
- 챕터/섹션/제목/상태 직접 편집
- **연결 요구사항 multiselect** (ID + 명칭 표시, 쉼표 오타 X)
- **✨ AI 추천** 버튼: 제목 기반으로 LLM이 요구사항 자동 선택
- **📑 선택된 요구사항 상세** expander: 정의/세부내용 확인하며 선택

## 📊 요구사항 목록 탭

### 커버리지 리포트 (상단 metric)
- 전체 / 커버됨 / 미커버 / 커버리지 % + progress 바
- **목표: 100%** (미커버 = 평가 감점 직결)

### 📊 카테고리별 커버리지
- 기능/성능/보안/품질 등 카테고리별 충족률
- 미커버 많은 카테고리 우선 정렬

### ⚠️ 미커버 요구사항 상세
- ID/카테고리/명칭/정의 표시
- **🔄 자동 매핑 시도**: 미커버 요구사항만 LLM이 적절한 슬라이드에 추가

### 전체 요구사항 표
- 기존 컬럼 + **연결 장표** 컬럼 (어느 슬라이드에 매핑됐는지 / 미매핑 표시)
- 정렬·검색 가능 (Streamlit 기본)

### ⬇️ 엑셀 내보내기
- 3시트(사업 개요 + 요구사항 + 장표 목차) 다운로드
- 외부 편집 후 사이드바에서 재업로드 가능

## 💡 기능 아이디어 도출 탭 (Agentic RAG)

### 언제 쓰나
- 제안서의 **핵심 전략·기술 아이디어**를 깊이 발전시키고 싶을 때
- 외부 시장 정보·경쟁사 사례·유사 사업 결과가 필요할 때
- 회사 보유 자료(이전 제안서·인증·실적)와 결합해 아이디어 구체화

### 사용 흐름

**1. 새 주제 만들기 (왼쪽 패널)**
- 주제명 입력 (예: "RAG 검색 정확도 고도화")
- 연결 요구사항 선택 (LLM이 매 답변에 활용)
- **➕ 주제 만들기** 클릭

**2. AI 초기 분석**
- 자동으로 첫 메시지 생성 — 3~5개 접근 방향 + "어디부터?" 질문
- 평가위원 관점, 사업 도메인 특성 반영

**3. 채팅으로 아이디어 발전**
- 자유롭게 질문/의견 입력
- AI가 매 턴 3가지 중 자율 선택:
  - 💭 **그냥 답변** (기존 컨텍스트로 충분)
  - 🔍 **웹 검색** (최신 시장/경쟁사/사례 필요 시)
  - 📎 **문서 요청** (회사 자료 필요 시 → 인라인 PDF 업로더 표시)
- 실시간 단계 표시:
  - `🧠 의사결정 중...`
  - `🔍 웹 검색 중...`
  - `✏️ 답변 생성 중...`
- 인용 출처 자동 추출 (URL + 제목)

**4. 자동 압축**
- 메시지 12개 초과 시 오래된 부분 자동 요약
- 첫 메시지 + 최근 4개 + 압축 요약본 = LLM이 보는 컨텍스트
- 요구사항/사업 개요는 항상 별도로 유지 (압축 영향 X)

**5. 🎯 아이디어 정리**
- 채팅 전체를 구조화된 마크다운으로 압축:
  - 핵심 아이디어 / 차별화 포인트 / 구현 방안 / 외부 참고 / 회사 자료 / 리스크 / 호소 포인트
- 정리본은 **편집 가능** + 다운로드 가능
- 정리된 주제만 전략 수립에서 선택 가능

### 설정 (왼쪽 ⚙️)
- 🔍 웹 검색 토글 (끄면 LLM 지식만으로 답변, 빠르고 저렴)
- 연결 요구사항 추가/제거
- 🗑️ 주제 삭제

### 권장 흐름
- 1주제당 **5~10턴**에서 정리하기
- 너무 길어지면 새 주제 만들어 분리
- 정리본은 전략 수립의 "💡 활용할 아이디어"에 자동 노출

## 🎯 전략 수립 탭

### 좌측: 설정 패널

**챕터 선택** + **하위 목차 expander** (확인용)

**요구사항 선택**
- 챕터에 매핑된 요구사항: 체크박스 (기본 전체 선택)
- 추가 요구사항: 챕터 외 요구사항 추가 가능 (multiselect)

**라운드 수**: 2 (권장) / 3

**💡 활용할 아이디어** (정리된 주제만 노출)
- 선택한 주제의 정리본이 토론 컨텍스트로 자동 첨부
- 전략가/비평가가 아이디어 알고 토론

**🎯 전략 수립 시작** 클릭 → `initial_input` 단계로

### 우측: 토론 영역

**단계별 흐름**:

```
initial_input
  └─ 사전 의견 + 참고 PDF 첨부 (선택)
       ↓
strategist → critic → ... (N라운드)
  └─ 중간에 [NEED_INFO]/[NEED_DOC] 발생 가능
       ↓
user_review
  └─ 추가 의견 + 참고 PDF 첨부 (선택)
       ↓
summarizing (최종 전략 정리)
       ↓
checking_docs (필요 문서 자동 식별)
  └─ 식별된 문서 업로드 큐 처리
       ↓
done (장표별 초안 작성 영역)
```

**참고 문서 업로드** (모든 단계 가능):
- 텍스트 추출 (자동, 빠름)
- 또는 페이지 입력 → **VLLM 파싱** (이미지·표·도표까지)
- 진행률 바 표시 (배치 병렬 처리)

### 장표별 초안 작성 (done 상태)

**🎯 챕터 전체 outline 일괄 생성** (권장 첫 단계)
- 챕터의 모든 슬라이드 outline을 한 번에 생성
- LLM이 슬라이드끼리 영역 침범 안 되게 자동 분배
- 기존 sections는 새 outline에 있는 제목만 유지

**슬라이드별 expander** (각 슬라이드)
- **목차 생성/재생성**: 개별 outline 생성 (같은 챕터 다른 슬라이드 정보도 컨텍스트로 활용)
- **⚡ 미작성 소제목 병렬 일괄 생성**: 작성 안 된 소제목들을 동시에 4개 워커로 병렬 생성
- **각 소제목**:
  - 📐 scope 표시
  - **AI 작성 / 재생성** (스트리밍)
  - **지우기**
  - **편집 가능한 text_area** (직접 수정 가능, 즉시 저장)

**📥 챕터 마크다운 다운로드** (1개 이상 작성 시 표시)
- 사업 개요 + 챕터 구성·요구사항 + 최종 전략 + 전략 논의 + 참고 문서 + 장표별 초안
- 완전한 컨텍스트 패키지 → Claude/GPT에 "PPT 만들어줘" 요청 가능

## 💡 모범 사용 흐름

### 시나리오 A: 빠른 초안 (1챕터 30분)
1. PDF 업로드 → 자동 파싱
2. 전략 수립 탭에서 챕터 선택
3. 사전 의견 간단히 입력 → 토론 시작 (2라운드)
4. user_review 단계 건너뛰기
5. 챕터 전체 outline 일괄 생성
6. 슬라이드별 병렬 일괄 생성
7. 마크다운 다운로드

### 시나리오 B: 깊이 있는 작성 (1챕터 1~2시간)
1. PDF 업로드 → 자동 파싱
2. 사업 개요 확인·편집
3. 목차 구성 탭에서 슬라이드·요구사항 매핑 점검 (커버리지 100% 확보)
4. 💡 아이디어 도출 탭에서 핵심 주제 2~3개에 대해 깊이 토론
   - 웹 검색으로 시장 정보 확보
   - 회사 자료 업로드해 구체 사례 확보
   - 각 주제 "🎯 아이디어 정리"
5. 전략 수립 탭에서 챕터 선택 + 활용할 아이디어 선택
6. 사전 의견 입력 → 3라운드 토론
7. user_review에서 추가 자료/의견 보강
8. 챕터 outline 일괄 생성 → 슬라이드별 본문 작성·편집
9. 마크다운 다운로드

### 시나리오 C: 협업 (여러 명이 분담)
1. 한 사람이 RFP 파싱 + 사업 개요 확정 → 엑셀 다운로드
2. 각자 엑셀로 로드 → 자기 챕터 작업
3. 작업 중 세션 JSON 저장으로 보관
4. 마크다운/JSON 공유

## ⚠️ 주의사항

### 보안
- **`.env` 파일 절대 공유 금지** — API 키 노출 시 무단 사용 위험
- **민감한 RFP**는 사용 전 검토 — PDF 내용·전략·아이디어가 OpenAI로 전송됨
- 채팅의 웹 검색도 검색 쿼리에 사업 정보 일부 노출 가능 (필요시 검색 토글 OFF)

### 비용 관리
- VLLM 파싱 (28페이지 기준 ~$0.5~1)
- Agentic 채팅 1턴 (검색 포함 시 ~$0.05~0.2)
- 챕터 1개 전체 작업 ~$2~4
- **OpenAI usage 페이지 주기적 확인 권장**

### 데이터 보존
- Streamlit 세션은 **브라우저 닫으면 사라짐**
- **30분 이상 작업 시 사이드바 → 💾 세션 → 저장** 권장
- 또는 엑셀 다운로드로 백업

### 정확도
- 자동 추출 결과는 **반드시 사람이 검토**
- 특히 슬라이드 ↔ 요구사항 매핑은 100% 신뢰 X → 목차 구성 탭에서 점검
- 사업 개요도 자동 추출 후 편집 권장
- LLM 생성 본문은 사실 검증 후 사용

## 🛠️ 문제 해결

| 증상 | 원인·해결 |
|---|---|
| 브라우저가 안 열림 | 콘솔에 표시된 주소(`http://localhost:8501`) 수동 접속 |
| "Already configured" 후 실행 이상 | `python` 폴더 삭제 후 재실행 (자동 재설치) |
| 파싱 결과 부정확 | 사이드바 사업 개요 직접 편집, 목차 구성 탭에서 슬라이드 편집 |
| VLLM이 빈 결과 반환 | 페이지 범위 축소 (예: 28p → 5p씩 시도), 콘솔에 `[VLLM TOC] 배치 N` 로그 확인 |
| 토큰 한도 에러 | 사업 개요 너무 김 → 편집해서 줄이기, 또는 참고 문서 업로드 안 하기 |
| API 키 에러 | `.env` 파일에 키 올바르게 입력했는지 확인, 앱 재시작 |
| 채팅 응답 실패 | 메시지 너무 김 → 새 주제 만들기, 또는 정리하기 후 재시작 |
| 세션 잃어버림 | 다음부터 💾 세션 저장 습관화 |

## 📚 산출물

작업 완료 시 얻을 수 있는 산출물:
1. **세션 JSON** (사이드바 💾) — 전체 작업 상태 백업/공유
2. **엑셀** (요구사항 탭 ⬇️) — 사업 개요 + 요구사항 + 목차 3시트
3. **챕터 마크다운** (전략 수립 탭 done 상태 📥) — 챕터별 완전한 패키지 (전략 + 초안 + 참고)
4. **아이디어 정리본** (아이디어 탭 ⬇️) — 주제별 구조화 마크다운

이 산출물들을 **Claude / GPT 같은 다른 LLM에 직접 붙여넣고 "PPT 만들어줘"** 하면 슬라이드 변환까지 자동화 가능.
